2012년 5월 1일

함수 이름이나 변수명 같은 식별자들을 길고 자세하게 (Objective-C처럼) 쓰면 코드가 읽기 쉬워질 것 같은 생각이 든다. 하지만 잘 생각해보면 코드 가독성에서 분량은 웬만해서는 다른 모든 요인을 거의 항상 압도한다. (예외적인 경우는 논문을 읽어야만 이해할 수 있는 알고리즘을 담은 코드 같은 것들이다.) 그리고 식별자는 코드에서 중복되는 것들을 일반화하고 이름으로 불리기 위해 반복되는 유일한 코드 뭉치이다. 그래서 식별자가 길어지면 당연히 코드의 분량도 늘어나게 되고, 코드의 가독성도 떨어진다. 물론 i보다는 index가 더 낫지만, 그렇다고 해서 positionOfString이 index보다 더 낫지는 않다고 생각한다.
